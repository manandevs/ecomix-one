3. Access Role in Your Application
You can now access the role from the session token in your components:13

typescript
import { auth } from '@clerk/nextjs/server'

export default async function Page() {
  const { sessionClaims } = await auth()
  const userRole = sessionClaims?.metadata?.role
  
  // Use the role for authorization
  if (userRole === 'admin') {
    // Admin-only content
  }
}

Or in client components:4

typescript
'use client'
import { useUser } from '@clerk/nextjs'

export default function ClientComponent() {
  const { user } = useUser()
  const userRole = user?.publicMetadata?.role as Role | undefined
  
  return <div>Your role: {userRole}</div>
}
This implementation ensures your custom role system integrates properly with Clerk's session tokens and metadata system, with the manager role constraint enforced at the API level.
